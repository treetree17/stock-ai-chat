# 项目报告：智能股票投资 AI 助手

## 1. 引言（Introduction）

### 1.1 项目构想 (Project Idea)
随着全球金融市场的日益复杂化与信息爆炸，个人投资者在面对海量的市场数据、实时滚动的财经新闻以及冗长专业的企业财报时，往往难以在短时间内快速提取有价值的关键信息并做出明智的投资决策。为了解决这一行业痛点，我们构想并设计了一个基于大语言模型 (LLM) 的智能股票投资 AI 助手。该助手旨在模拟专业金融分析师的日常工作流程与研判逻辑，结合实时的量化行情指标、最新的互联网财报新闻以及深度本地研报，为用户提供一站式、自然语言交互的智能投资分析咨询服务，彻底打破普通投资者与专业机构之间的信息壁垒。

### 1.2 解决方案的关键组成部分 (Key Components of the Solution)
本系统的解决方案主要由以下几个核心技术与业务组件构成：
1. **意图识别与实体提取模块**：系统前端接入后，会首先自动识别用户的提问意图（精准判断是否为股票金融相关的垂直领域问题），并从自然语言中提取具体的股票实体信息，以便后续精准调取数据。
2. **多源数据并发抓取网络**：
   - **量化行情源 (YFinance)**：实时获取股票当前的最新交易价格、开盘价、涨跌幅动态以及 K 线图表的基础配置数据，提供客观的数据支撑。
   - **实时外部资讯 (Google News RSS)**：在用户提问的瞬间，在线并发抓取最具时效性、最新发生的几条全球新闻短讯，捕捉市场情绪。
   - **本地深度化知识库 (RAG Vector DB)**：通过 SQLite 存储预先处理好的深度企业财报和专业分析文章的向量。系统根据用户的问题，运用余弦相似度检索深度背景事实。
3. **大模型推理引擎 (LLM Service)**：作为系统的大脑，整合上述所有的多维度上下文信息，构建高度定制化的 Prompt，指导大语言模型流式输出逻辑严密、论证充分的金融分析结论。

### 1.3 主要成果总结 (Summary of Key Findings)
通过本项目的完整开发与多轮测试，我们成功构建了一个轻量级但功能完备的 MVP（最小可行性产品）。系统能够在仅仅几秒内完成从意图识别到多源并发检索，再到大模型结合研报与实时新闻生成解答的全过程业务流。测试结果表明，通过创新性地引入 RAG（检索增强生成）机制与实时联网查询管线，模型彻底摆脱了传统回答中的幻觉现象与知识库过期的严重限制，不仅能精准作答近期的突发财报事件，还能提取本地导入的专业长篇研报给出极具深度的见解，显著提升了垂直领域金融问答的质量与可靠性。

## 2. 方法论（Methodology）

### 2.1 解决业务问题的方法 (How the Business Problem is Solved)
本系统采用的是多智能体协作与 RAG 融合架构来解决投资者面临的严重信息过载与认知局限问题。传统的 LLM 具有信息滞后和缺乏具体企业私密研报的问题，我们通过设计以下管线（Data Flow）来解决这些固有难题：

1. **动态意图拦截机制**：系统首先对用户输入的自然语言进行轻量级的意图分析。如果在本地意图词库匹配到该问题并非股票金融相关（如普通的日常问候），系统将绕过后端复杂的金融数据查询流程，直接进入基础对话模式。这极大地节省了系统计算资源与外部 API 的调用成本。
2. **并发式数据流体系 (Concurrent Data Pipeline)**：当检测到明确的股票金融业务意图时，后端 (基于 FastAPI) 利用 Python 的 \syncio\ 异步机制实现多路并发的金融数据采集：
   - 同步通过 YFinance API 获取精确的客观交易数字与行情指标；
   - 同步触发 \LiveNewsService\ 进行时事新闻的轻量化爬取，确保新闻的一手性；
   - 同步将用户查询词通过本地 FastEmbed 模型计算 Embedding 向量，进入本地知识库 (SQLite) 做近似近邻搜索 (KNN/Cosine Similarity)，找出预摄入 (Ingest) 好的核心财报段落。
3. **Prompt 组装与情景注入**：将抓取到的外部短讯、内部研报段落、股票基础面数据以及用户的历史交互对话，无缝拼接为结构化的系统提示词 (System Prompt)。这一过程迫使 LLM 扮演专业金融分析师的角色，进行阅读理解、信息交叉验证与综合归纳。

### 2.2 核心挑战与克服策略 (Major Challenges and Strategies)

**挑战一：大模型信息滞后与幻觉现象**
纯粹依赖原生大参数模型提供的回答，极易生成已经过期（比如一年前的旧数据）或者凭空捏造的虚假股价信息，这对容错率极低的金融投资场景是致命的。
*克服策略*：我们引入了双轨制的新闻与知识机制体系。对于最新的几小时内的新闻，我们利用 \LiveNewsService\ 突破无网络限制，从外部实时抓取最新动态；对于长线深度认知，如企业年报和长篇研报，我们离线开发了数据处理脚本，将长篇大论切片 (Chunking) 转化为高维向量存储在本地 SQLite 中，通过 RAG 的精准召回直接作为模型的参考事实（Ground Truth）。这两者完美互补，从根本上根治了时间滞后性与幻觉。

**挑战二：复杂检索导致的系统高延迟**
如果在回答用户提问前，系统需要依次顺序拉取不同新闻源、查行情以及进行向量计算，整个链路将耗时十几秒甚至更长，这将导致极差的用户体验。
*克服策略*：使用 FastAPI 的异步（Asyncio）特性重构核心调度逻辑。我们将股票行情获取、外部新闻拉取、文本向量化三大 IO 密集型操作放到 \syncio.gather\ 中并发执行。这一架构优化把响应时间的瓶颈降到仅仅是最慢的那一个单一服务的响应时长，成功将系统总延迟压缩至原来的三分之一以下。

**挑战三：RAG 向量检索效率与本地部署的权衡**
引入庞大的商业级向量数据库（如 Milvus / Chroma）会极大增加项目的部署负担、占用过高内存，并提高系统的维护门槛。
*克服策略*：采取降维打击的思路，我们采用了最传统的 SQLite 搭配本地轻量级 \astembed\ 模型的融合方法。将生僻的高维向量序列化成 JSON 存在关系型数据库的长文本字段中，并且在应用层手写了基于内存的余弦相似度比对算法。在保持百万级别字符知识库查询在百毫秒返回的同时，做到了真正的零外部数据库安装开销，极大提升了部署便捷性。

## 3. 结果（Results）

经过详尽的测试演练，本项目成功达成了以下核心应用结果：

1. **金融知识图谱构建与检索表现**：通过本地 RAG 预处理与在线新闻抓取的默契配合，在面对复杂的研判问题时，系统能够完美规避原生大模型的凭空臆想。取而代之的是，它能够非常准时、犀利地提取出导入知识库中的细分论点（如企业的云业务增长率、杠杆水平等事实），给出极具参考价值的回答。
2. **多模态数据展示与流畅渲染**：系统不仅提供了纯文本的深度交互与分析，当用户查询具体的某只股票时，系统能在给出文本流式输出 (SSE Streaming 的动态打字机效果) 的同时，由前端自动化渲染可视化的股票 K 线走势分析图，形成了宏观面分析-图表展示-走势研判的完整交互应用闭环。
3. **架构的高鲁棒性与脱耦设计**：历经核心架构优化，项目已全面向基于 Yahoo Finance（量化数字基准）与谷歌新闻（文字基本面）的稳定并发流通道靠拢。这一脱耦设计极大提升了项目架构的健壮性与抗压性，确保系统在复杂网络环境下依然能稳定提供高质量的分析预测服务。

## 4. 局限性和未来工作（Limitations and Future Work）

**主要局限性：**
1. **本地检索的算力瓶颈**：当前手写的 SQLite 余弦解析检索法，虽然轻量且极易部署，但一旦研报数量呈现指数级突增，线性搜索将会带来一定的计算瓶颈与查询耗时增长。
2. **单次推理的业务天花板**：当前模型的推理仍属于一次过的指令式生成，尚不支持复杂的多步自主规划能力。系统暂时缺乏像 ReAct (Reasoning and Acting) 框架那样能够反复自我检索、自我验证并纠错的能力。

**未来工作与迭代规划：**
1. **平滑升级专业型数据库引擎**：拟向 PostgreSQL 结合 \pgvector\ 扩展架构演进，用于代替当前的存算一体方式，以此突破大规模研报检索的算力限制，实现生产级的高可用。
2. **多智能体 (Multi-Agent) 的大一统编排**：计划引入业界成熟的 LangChain 或相关 Agent 框架，把当前的资讯爬虫模块和行情源模块注册为真正的智能体工具。赋予底层模型自主分析的规划能力，使其成为真正的高端金融研究 AI。

## 5. 工作分配（Work Allocation）

本项目采用高度敏捷的团队交叉协作分工模式执行，各成员所负责的核心任务板块如下：

- **组员 A（后端架构设计与 LLM 核心调优）**：
  负责高并发 FastAPI 主框架的搭建与 RESTful 接口设计；实现 RAG 检索体系的大模型推理集成，包括主导编写本地向量化与余弦相似度比对算法；统筹多协程调度的流式并发网络问答机制。
- **组员 B（多源数据接入与数据工程化）**：
  全面负责外部异构 API 的标准化集成，主导行情数据的对接以及实时网页新闻流的云端抓取；并负责本地离线长文本文件智能切块并注入向量知识库的自动化脚本开发工作。
- **组员 C（前端可视化呈现与增强交互体验）**：
  应用原生 Web 技术搭建平滑抗抖动的流式问答交互页面；负责基于外部数据的股票金融图表的前端无缝动态渲染，以及整体系统 UI 响应式与体验级优化。
- **组员 D（数据底层管控与整体测试部署）**：
  负责底层轻量化数据体系的设计管理，主导系统的意图拦截策略与上下文记忆库的维系；全面负责项目的整体并发容量与模型回答准确性评测，以及最终项目文档资料、结题报告的校准与撰写。
